### 4.2 Стратегії оптимізації при масштабуванні

#### a) Async паралелізація
```python
import asyncio
from openai import AsyncOpenAI

# Замість послідовних запитів — паралельні батчі
async def process_batch(chats, batch_size=10):
    client = AsyncOpenAI()
    for i in range(0, len(chats), batch_size):
        batch = chats[i:i+batch_size]
        tasks = [analyze_chat(client, chat) for chat in batch]
        results = await asyncio.gather(*tasks)
```

| Кількість | Послідовно | Паралельно (10) | Прискорення |
|-----------|-----------|----------------|-------------|
| 120 | ~8 хв | ~1.5 хв | 5x |
| 1 000 | ~55 хв | ~10 хв | 5.5x |
| 10 000 | ~9 год | ~1.5 год | 6x |

#### b) OpenAI Batch API (для великих обсягів)
- **50% знижка** на вартість
- Результати доступні протягом 24 годин
- Ідеально для 1000+ діалогів

| Кількість | Звичайний API | Batch API | Економія |
|-----------|-------------|-----------|----------|
| 1 000 | $5.75 | $2.88 | 50% |
| 10 000 | $57.50 | $28.75 | 50% |

#### c) Кешування результатів
- Зберігати згенеровані діалоги локально
- При повторному запуску аналізу — не перегенеровувати
- Зберігати raw API-відповіді для дебагу

#### d) Prompt caching (OpenAI)
- OpenAI автоматично кешує довгі промпти (>1024 токенів)
- Кешовані input-токени коштують **50% дешевше**
- При аналізі 120+ діалогів з однаковим system prompt — значна економія

## 5. Rate Limits

### Стандартні ліміти OpenAI (Tier 1)

| Модель | RPM (запити/хв) | TPM (токени/хв) |
|--------|----------------|-----------------|
| GPT-4o | 500 | 30 000 |
| GPT-4o-mini | 500 | 200 000 |

### Стратегія роботи з лімітами

| Обсяг | Стратегія |
|-------|-----------|
| 120 діалогів | Без обмежень — вкладаємось в ліміти послідовно |
| 500 діалогів | Async з batch_size=10, delay 0.5s між батчами |
| 1 000+ | Async з адаптивним batch_size + exponential backoff |
| 10 000+ | OpenAI Batch API + розбиття на кілька запусків |

```python
# Exponential backoff для rate limits
import time

def call_with_retry(func, max_retries=3):
    for attempt in range(max_retries):
        try:
            return func()
        except RateLimitError:
            wait = 2 ** attempt  # 1s, 2s, 4s
            time.sleep(wait)
    raise Exception("Max retries exceeded")
```

## 6. Рекомендації

### Для хакатону (120 діалогів)
- Використовувати **Варіант A** (mini + 4o)
- Послідовна обробка з `tqdm` прогрес-баром

### Для продакшену (1000+ діалогів)
- Async-обробка з паралельними батчами
- OpenAI Batch API для некритичних задач
- Моніторинг витрат через OpenAI Dashboard
- Логування всіх API-відповідей для аудиту

### Для великих масштабів (10 000+)
- Перехід на GPT-4o-mini для аналізу з fine-tuned моделлю
- Batch API + розбиття на добові запуски
- Розглянути self-hosted open-source моделі (Llama, Mistral) для генерації
- Кешування та дедуплікація промптів
